- **Skills hook cache — done** (ccstatusline parity): `src/data/skills-cache.ts` + `--hook` mode record every skill invocation (PreToolUse:Skill **and** UserPromptSubmit `/slash`) to a per-session JSONL under `$XDG_CACHE_HOME/cc-status-dash/skills/`; `mergeSkills()` folds the cache into the transcript-derived list (dedupe keep-most-recent-last) so the `skills` widget is accurate and compaction-proof, degrading to the transcript scan when no hook is installed. `/setup` optionally registers the managed hooks. Path-traversal-guarded, control-char sanitized, best-effort (never throws/prints).
- **Independent re-audit + 6 small parity gaps closed**: spawned a fresh audit (not trusting our own PARITY.md, since it had already been caught stale once this session) that read ccstatusline/Claude HUD source directly. Closed: **widget `merge: "no-padding"`** (closes the padding gap at a merge seam, not just the separator — ccstatusline `WidgetItem.merge` parity, via a two-pass `buildLineWidgets` that looks ahead at the next widget's merge value); **`config-counts` `threshold`** (suppresses the line below N total items — Claude HUD `environmentThreshold` parity); **`gitCacheTtlSeconds`** (configurable git-provider disk-cache TTL, was hardcoded 2s); **`custom-command` `timeout` + `preserveColors`** (per-command timeout override; lets the command's own ANSI drive coloring instead of being stripped — ccstatusline `CustomCommand` parity); **`model`/`provider` `override`** (manual label takes full precedence over auto-detection, for custom proxies/gateways — Claude HUD `modelOverride`/`providerName` parity; also fixed a stale, never-consumed `prefix` option on the `provider` TUI spec). +16 tests. (Git branch overflow mode was investigated and found to be already covered by the existing generic `maxWidth`/`autoWrap` primitives — not implemented as a separate option to avoid redundancy.)
- **flexMode (effective-width policy) — done**: ccstatusline parity, confirmed by reading its source (`renderer.ts`) — flexMode is **not** a grid engine, it's a 3-way width-reservation policy applied to auto-wrap and flex-fill: `"full"` trims a small (6-col) margin, `"full-minus-40"` trims a large margin (room for Claude Code's own wider input box), `"full-until-compact"` switches between the two once context usage crosses `compactThreshold` (default 60%). Unset keeps the raw terminal width (prior behavior, no config = no change). Wired through `effectiveWidth()` in the renderer, used by both auto-wrap and flex-separator fill; clamped to never go negative. Editable in the TUI global screen; valibot-validated. +6 tests. This — together with flex separators and the panel style — closes the "flexMode/auto-align (full layout engine)" item that was previously over-scoped as a large CSS-grid build; a true multi-column grid layout remains a distinct, larger, and still-deferred idea with no ccstatusline precedent to port.
- **Widget icon toggle — done**: global `icons: false` hides decorative widget glyphs (leading icons like `✱ ✦ ⚙ ⏱ ◱ ✉`) while keeping structural glyphs (separators `│`, pace/ahead-behind arrows `↑↓⇡⇣`, on/off `◉○`, bars, agent spinners). Implemented via a dedicated `ic()`/`icp()` helper (audited all 66 `sym()` sites, converting only the ~28 decorative ones); hides cleanly through `lv()` (label→value-only) and `icp()` (no stray leading space). Editable in the TUI global screen; valibot-validated; default on.
- **Panel render style — done**: a 4th line `style: "panel"` frames the line's content in a box (Claude HUD panel look) — one config line → three rows, charset-aware (rounded `╭─╮` vs ASCII `+-+`), border painted via the painter (honors NO_COLOR/colorDepth), equal-width rows, empty lines still culled. Cyclable in the TUI (`s`), valibot-validated. (The broader CSS-grid/multi-column layout remains deferred; this delivers the framed-panel look.)
- **Flex separators (auto-align) — done**: new `flex-separator` widget + renderer support on inline lines. A flex spacer fills the remaining terminal width so trailing widgets right-align; multiple spacers split the gap evenly (center/spread layouts), with a configurable `fill` glyph. No separator is drawn adjacent to a spacer, and content wider than the terminal degrades to no fill (never overflows/crashes). Tested at fixed widths.
- **Install-to-settings.json — done**: `src/config/install.ts` writes the `statusLine` block (and, opt-in, the managed skills hooks) into Claude Code's `settings.json`. Pure `buildSettings()` deep-merges — every unmanaged key preserved, our own hooks strip-then-re-added so re-installs are idempotent, toggling hooks off cleanly removes them. IO wrapper backs up (`.bak`), writes atomically, refuses to clobber invalid JSON, never throws. Exposed as **`--install [--install-hooks] [--dry-run]`** and as a **TUI overlay** (`i` key, `h` toggles hooks). $CLAUDE_CONFIG_DIR-aware; command auto-detected.
- **Deferred (need-unavailable-data)**: live usage API + weekly-opus/sonnet (needs Anthropic OAuth), Jujutsu/i18n (per original plan). No layout items remain deferred — flexMode was confirmed (by reading ccstatusline's source) to be a width policy, not a grid engine, and is now implemented.
- **Compaction `byTrigger`/`tokensReclaimed` breakdown — done** (ccstatusline `compaction.ts` parity): `data/transcript.ts` now reads `compactMetadata.trigger` (auto/manual, else "unknown" — never guessed, sidechain-excluded via the existing `isSidechain` skip) and sums `max(0, preTokens-postTokens)` across every `compact_boundary` marker. `compaction-counter` gained `showTriggers`/`showReclaimed` toggles rendering `⟳ 2 (1 auto, 1 manual) ↓180.0k`. +9 tests; verified end-to-end against a synthetic transcript.
- **Timer `timeFormat` elapsed modes — done** (Claude HUD `timeFormat` parity): `reset-timer`/`weekly-reset-timer` gained a `timeFormat: "relative" | "absolute" | "both" | "elapsed" | "elapsedAndAbsolute"` option — `relative` is the existing countdown (default, unchanged); `absolute` is the wall-clock reset time alone; `both` combines them (`1d 6h, 05:30`); `elapsed`/`elapsedAndAbsolute` are a *different concept* — % of the window that has passed (`100% elapsed`), not time remaining. Takes priority over the legacy `timestamp`/`hoursOnly` options when set; unset `timeFormat` = zero behavior change for existing configs. +6 tests; verified live at both widths.
- **HUD label alignment — done** (Claude HUD `alignLabels`/`label-align.ts` parity, generalized): global `alignLabels: true` right-pads every inline/panel line's leading `lv()`-style label to the widest such label across the whole config, so values on separately-stacked lines start at the same column (`Ctx 46%` / `5h  38%` / `7d  61%`). Generalized from HUD's hardcoded context/usage/weekly trio to *any* label-bearing line — our widget set is far broader than HUD's fixed three. Required restructuring `render()` into two passes (build all lines, then align, then render each to its final string) so the alignment pass can see every line's built widgets before any style-specific rendering happens. The detection heuristic precisely matches `lv()`'s contract (trailing space + a following non-empty segment) rather than a bare `color === "label"` check, which a first draft got wrong — that broader check also matched widgets whose own *value* happens to default to the "label" color (e.g. `custom-text`) and a `dim:"parens"` wrapper's parenthesis segments; caught and fixed via test before shipping. +4 tests; verified against real progress-bar widgets in-process.
- **Remaining from the independent re-audit (medium/large, not yet implemented)**: **Powerline color-cycling themes** — ccstatusline's separate depth-aware 10-palette system that auto-cycles per segment position (`utils/colors.ts`), distinct from our role-based theme system; would need new palette data + a TUI selector. This is the last item from the audit.
